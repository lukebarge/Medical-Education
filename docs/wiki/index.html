1. Architecture Overview

The library is structured in five layers. Higher layers depend on lower ones; the medical examples sit at the top and consume everything below.

🫀
examples/
Medical animations (HeartBeat, Breathing…)
🎛️
controls/
Slider, Toggle, Button, ControlPanel
✨
effects/
ParticleSystem, FlowPath, WaveForm
🔷
shapes/
Circle, Rect, Path, Text
⚙️
core/
Canvas, Animation, Scene, Tween…
User Input (DOM)
    │
    ▼
Controls (Slider, Button…) ──emit('change')──▶ Binding ──▶ Animation property setters
                                                                   │
                                                                   ▼
                                                        Animation._update(dt)
                                                        Shape.animate() / Effects.update(dt)
                                                                   │
                                                                   ▼
                                                        Animation._draw(ctx)
                                                                   │
                                                                   ▼
                                                           Canvas.ctx → browser

Every class extends EventEmitter, enabling pub/sub communication. Two animation loop patterns are provided: Animation (single object, own RAF loop) and Scene (multiple drawables sharing one loop).

2. Quick Start

import { Canvas }       from './src/core/Canvas.js';
import { HeartBeat }    from './src/examples/HeartBeat.js';
import { ControlPanel } from './src/controls/ControlPanel.js';
import { Slider }       from './src/controls/Slider.js';

// 1. Create canvas (HiDPI-aware)
const canvas = new Canvas('#app', { width: 800, height: 600 });

// 2. Create animation
const heart  = new HeartBeat(canvas, { bpm: 72, contractility: 0.8 });
heart.play();

// 3. Wire up controls
const panel = new ControlPanel('#controls', { theme: 'dark' });
panel.add(new Slider({ label: 'BPM', min: 30, max: 200, value: 72,
  onChange: v => heart.setBPM(v) }));
panel.mount();

// 4. React to events
heart.on('systole', () => console.log('Systole!'));

📡 EventEmitter
Minimal pub/sub base class inherited by all library classes. Enables decoupled communication between components.

Methods

emit(event, data)string, anythisFire all listeners for an event with optional payload.

Abstract base class for single-object animations with a built-in requestAnimationFrame loop. Subclass and override _update(dt) and _draw(ctx).

🔗 Binding utility (static factory)
Creates two-way bindings between a control and an animation property. Control changes update the target; target emitting ${property}Changed updates the control.
// Link slider to heart.bpm
Binding.create(bpmSlider, heart, 'bpm');
// Now: slider change → heart.bpm = value
//      heart.emit('bpmChanged', value) → updates slider
